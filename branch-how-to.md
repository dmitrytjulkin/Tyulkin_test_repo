# Работа с ветками в Git

Ветки позволяют работать над разными версиями проекта одновременно.
Это полезно для разработки новых функций без влияния на основную версию.

## Полная инструкция по работе с ветками

### Как создать ветку
1. Создать новую ветку: `git branch имя_ветки`
2. Или создать и сразу переключиться: `git checkout -b имя_ветки`

### Как переключиться в ветку
1. Переключиться на существующую ветку: `git checkout имя_ветки`
2. Посмотреть список всех веток: `git branch`
3. Посмотреть удаленные ветки: `git branch -r`

### Как отправить ветку на сервер
1. При первом push: `git push -u origin имя_ветки`
2. Последующие push: `git push`
3. Просмотреть все ветки на сервере: `git branch -a`

### Полезные команды
- Слияние веток: `git merge имя_ветки`
- Удаление ветки: `git branch -d имя_ветки`
- Удаление ветки на сервере: `git push origin --delete имя_ветки`
